# STREHË Package Pricing Draft

## Purpose

This document turns the regional pricing benchmark into a first practical package model.

It is a draft, not a final public pricing page.

The goal is to define:

- package names
- monthly price ranges
- visit frequency
- what is included
- what is not included
- which client each package fits best

---

## 1. Pricing principle

STREHË should not try to be the cheapest option.

It should be:

- believable for the region
- sustainable for operations
- clear enough for clients to understand
- disciplined enough to avoid founder burnout

The service should be priced above raw cleaning/help because it includes:

- trust
- access responsibility
- scheduling
- reporting
- follow-up
- communication from abroad

---

## 3. Package 1: Essential Check

### Draft price

- `€69/month`

### Included

- 1 scheduled apartment visit per month
- basic visible-condition check
- basic access/readiness check
- short update after visit
- photos when useful
- note of any visible issue

### Best-fit client

This package fits owners who:

- live abroad
- want basic reassurance
- do not need frequent follow-up
- mainly want to know the apartment is not ignored

### Not included

- frequent coordination
- urgent response guarantee
- regular errands
- detailed preparation before arrival
- ongoing technician management

### Operational note

This should stay very disciplined.

If clients start requesting frequent extra tasks, they should move to a higher package or pay separately.

---

## 4. Package 2: Care Plus

### Draft price

- `€109/month`

### Included

- 2 scheduled apartment visits per month
- visible-condition checks
- basic readiness checks
- update after each visit
- photos when useful
- issue flagging
- limited local follow-up for simple apartment-related matters
- basic arrival/readiness awareness

### Best-fit client

This is likely the main Year 1 package.

It fits owners who:

- are abroad most of the time
- want more than a monthly check
- want clearer ongoing visibility
- want one local point of contact

### Not included

- heavy errand running
- 24/7 emergency promise
- large renovation supervision
- unlimited technician coordination
- cleaning unless separately arranged

### Operational note

This package should become the operational standard.

Most workflow, reporting, and app design should support this package first.

---

## 5. Package 3: Arrival Ready

### Draft price

- `€159/month`

### Included

- 2 scheduled apartment visits per month
- pre-arrival preparation support
- apartment readiness check before return
- airing / light preparation when appropriate
- issue reporting and local follow-up
- clearer arrival-related communication
- coordination of small readiness tasks

### Best-fit client

This package fits owners or families who:

- return to Kosovo periodically
- want the apartment ready before arrival
- need more active local attention
- value convenience and reduced stress

### Not included

- deep cleaning unless separately arranged
- large repairs or renovation management
- unlimited errands
- emergency response guarantee
- ongoing property rental management

### Operational note

This package is higher-touch.

It should be sold carefully so it does not become “do everything for me.”

## 6. Optional add-ons

Add-ons should stay simple at launch.

## Add-on table

| Add-on | Draft price | Notes |
| --- | ---: | --- |
| Extra visit | `€25–€35` | Useful for one-off checks or special timing |
| Pre-arrival readiness visit | `€35–€50` | If not included in package |
| Technician coordination | `€20–€40` per issue | For simple follow-up only |
| House interest registration | Free | Not a launch service, only future interest |
| Deep cleaning | Quote / partner cost | Should not be bundled unless process is ready |

### Add-on rule

Add-ons should not turn into unlimited concierge work.

If an add-on starts becoming frequent, it should either:

- become a defined package feature
- be priced separately
- or be refused if it does not fit the operating model

## 7. Public pricing recommendation

At launch, full pricing does not need to be published immediately.

Recommended public wording:

- “Packages start from around €69/month.”
- “The right setup depends on visit frequency and the support you need.”
- “Message us and we’ll suggest the most suitable option.”

### Why not publish everything immediately?

Because early STREHË still needs to learn:

- workload per apartment
- what owners ask for most
- how often issues appear
- how much follow-up is needed

But internally, the team should still use these package prices as planning anchors.

---

## 8. Boundaries that must be clear

Every package should make clear:

- visits are scheduled, not random on-demand
- emergency response is not guaranteed at launch
- large repairs are not included in monthly fee
- cleaning is separate unless explicitly arranged
- technician/vendor costs are paid separately by the owner
- houses are not part of launch scope

These boundaries protect trust.

They also protect the founder.

## 10. Financial implication

## If 30 clients average `€69`

Revenue:

- `30 x 69 = €2,070/month`

This is likely too low if staff are needed.

## If 30 clients average `€109`

Revenue:

- `30 x 109 = €3,270/month`

This is more realistic but still tight with two staff.

## If 30 clients average `€129`

Revenue:

- `30 x 129 = €3,870/month`

This begins to look better, but still needs careful staffing and cost control.

## If 30 clients average `€159`

Revenue:

- `30 x 159 = €4,770/month`

This is healthier, but only if the service level justifies the price and the workload is manageable.

### Key lesson

The business should not try to build a staffed operation on mostly low-entry packages.

It needs a healthy mix, with most clients ideally near the core package or above.

## 11. Recommended Year 1 package target

For Year 1, a good target mix might be:

| Package | Share of clients | Example with 30 clients |
| --- | ---: | ---: |
| Essential Check | 30% | 9 clients |
| Care Plus | 50% | 15 clients |
| Arrival Ready | 20% | 6 clients |

### Revenue example

- Essential: `9 x 69 = €621`
- Care Plus: `15 x 109 = €1,635`
- Arrival Ready: `6 x 159 = €954`

Total:

- `€3,210/month`

### Interpretation

This mix is probably still modest if STREHË needs two staff.

To support staff more comfortably, either:

- average package value must rise
- add-ons must contribute
- staffing must stay flexible/part-time
- service delivery must be very efficient

---

## 12. Revised staffing warning

With these more region-sensitive prices, two staff members at 30 clients may be possible but tight.

This should be read together with:

- `operating-cost-model.md`

If average revenue per client is around:

- `€100–€110/month`

then two staff can put pressure on margins.

If average revenue per client is closer to:

- `€130–€160/month`

then the staffing picture becomes healthier.

So STREHË should avoid:

- too many low-tier clients
- too many custom extras without charging
- full-time staffing before the revenue mix supports it

It should also avoid silently absorbing:

- repair costs
- technician fees
- extra visits
- cleaning
- long coordination work
- unpriced errands

---

## 13. Working recommendation

The first public pricing architecture should likely be:

- simple
- package-based
- not too expensive at entry
- clear that extra work costs extra

Recommended draft:

- Essential Check: `€69/month`
- Care Plus: `€109/month`
- Arrival Ready: `€159/month`

This should be tested against:

- real client reactions
- workload
- competitor perception
- staff cost reality

The numbers are not final, but they are a more believable starting point than the earlier high-level assumptions.
